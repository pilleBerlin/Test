git Befehle

  • git add //Diese Datei wird überwacht und dem Repository zur Verfügung gestellt
  • git commit -m "message" //Mach einen Snapshot und sichere es im lokalen Repository
  • git status //gibt den aktuellen Status zurück, bsp. was zu tun ist, welcher branch etc.
  • git push // alle Daten die markiert sind und commitet, werden an git gesendet
  • git pull //alle neuen Daten werden heruntergeladen
  • git log //zeigt alle Commitments an, mit einem Hashtag, eindeutig
  • git reset --hard || //Die Version wird auf die zurück gesetzt!

    • Konflikt bei push

  • Von "<<<<< HEAD" bis "=====" Zeichen ist dein Code, das nach dem "======" bis ">>>>>" ist der andere Code. Hier musst du entscheiden welcher Teil korrekt ist. Die Korrektur wird in der Datei durchgeführt, weil das Ergebnis des Konflikts in der Datei hinterlegt wird
